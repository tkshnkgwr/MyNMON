# システム仕様書 (SPEC.md)

本仕様書は、クロスプラットフォーム対応・超軽量CUIシステムモニター `rust-nmon` の機能要件、非機能要件、および内部構造を定義したものです。

---

## 1. システム概要

`rust-nmon` は、ターミナル上でローカルコンピュータのハードウェアリソースをリアルタイムに監視できるアスキーUI（CUI）ユーティリティです。
Linux の古典的な監視ツール `nmon` にインスパイアされ、以下の要素を重視して設計されています。

- **ポータビリティ**: Windows、Linux、macOS などのクロスプラットフォーム環境で変更なしにビルド・動作すること。
- **超軽量性**: CPUやメモリなどのリソース自体の消費を最小限に抑え、監視対象システムに不要な負荷を与えないこと。
- **インタラクティブ性**: キー入力によって動的に表示セクションのトグル切り替えができること。

---

## 2. システムアーキテクチャ

本システムは単一の実行可能バイナリであり、単一スレッドのイベントループモデルを採用しています。

```text
┌────────────────────────────────────────────────────────┐
│                      rust-nmon                         │
│                                                        │
│  ┌───────────────────────────┐                         │
│  │       Event Loop          │                         │
│  │  (1-second interval or)   │                         │
│  │  (Keyboard interrupt)     │                         │
│  └─────────────┬─────────────┘                         │
│                │                                       │
│                ▼                                       │
│  ┌───────────────────────────┐                         │
│  │   Metrics Collection      │                         │
│  │  (sysinfo crate)          │                         │
│  │  - CPU Cores, Memory      │                         │
│  │  - Disks, Networks, Proc  │                         │
│  └─────────────┬─────────────┘                         │
│                │                                       │
│                ▼                                       │
│  ┌───────────────────────────┐                         │
│  │     Terminal Rendering    │                         │
│  │  (crossterm crate)        │                         │
│  │  - Color styling          │                         │
│  │  - ASCII progress bar     │                         │
│  └───────────────────────────┘                         │
└────────────────────────────────────────────────────────┘
```

### 主要ライブラリと役割
- **`sysinfo`**: ハードウェア、プロセス、OS、ネットワーク、ディスクのメトリクスをOS非依存の抽象API経由で取得します。Windows APIおよびLinuxの `/proc` ファイルシステムなどのプラットフォーム固有の実装を内包しています。
- **`crossterm`**: 画面切り替え（オルタネートスクリーン）、カーソル非表示、rawモード制御、キーイベント入力の監視、およびターミナル制御シーケンスを実行します。

---

## 3. 機能要件

### 3.1 画面表示およびセクション制御
システムは以下の5つの情報セクションを持ち、それぞれキー入力により動的に表示/非表示（トグル）を切り替えられます。

1. **CPU セクション (`c` キー)**:
   - 個々のCPUコアの使用率 (%) を取得し表示。
   - 使用率に応じたアスキー進行バー（`[===>    ]`）の描画。
   - 負荷レベルに応じた自動着色（緑色: <= 40%、黄色: 40% < 負荷 <= 80%、赤色: > 80%）。
2. **メモリセクション (`m` キー)**:
   - 物理メモリ（RAM）の総容量、使用量、空き容量を GB 単位で表示。
   - メモリ使用率 (%) の表示と、アスキーバーの描画（85%を超えると赤色に着色）。
3. **ディスクセクション (`d` キー)**:
   - 各ディスクのマウントポイント、ファイルシステム形式（例: `"NTFS"`, `"ext4"`）。
   - 空き容量 / 総容量、および使用率 (%) の表示。
4. **ネットワークセクション (`n` キー)**:
   - 稼働中の各ネットワークインターフェース名。
   - 受信速度（Rx）および送信速度（Tx）を KB/s 単位で表示。
5. **プロセスセクション (`p` または `t` キー)**:
   - CPU使用率が高い順にソートされた上位8件のアクティブプロセス。
   - PID、プロセス名、CPU使用率 (%)、物理メモリ占有量 (MB) を表形式で表示。

### 3.2 アプリケーション制御キー
- `q` または `Esc` キーの入力により、ターミナルの状態（rawモード解除、オルタネートスクリーン離脱、カーソル再表示）を元通りに修復した上で、正常終了コード `0` で即座にプロセスを終了します。

---

## 4. 非機能要件

- **更新頻度**: 1秒（1000ms）間隔でメトリクスをリフレッシュし画面を再描画します。
- **キー割り込みレスポンス**: キー入力が発生した場合は、1秒のタイマー完了を待たずに即座に状態を変更して再描画を行います。
- **フットプリント**:
  - **実行ファイルサイズ**: 350 KB 以下（最適化リリースビルド時）。
  - **実行時メモリ使用量**: 25 MB 以下（通常のプロセス監視時）。
- **ロバスト性**:
  - パニック発生時はcrosstermによる画面破壊を避けるため、即座にプロセスを中断（`panic = 'abort'`）します。
  - OS情報の取得失敗やディスクが存在しない場合でも、フォールバック値（例: `"Unknown"`, `0`）を用いてプログラムがクラッシュせずに処理を続行します。
