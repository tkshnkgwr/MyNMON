# システム仕様書 (SPEC.md)

本仕様書は、クロスプラットフォーム対応・超軽量CUIシステムモニター `MyNMON` の機能要件、非機能要件、および内部構造を定義したものです。

---

## 1. システム概要

`MyNMON` は、ターミナル上でローカルコンピュータのハードウェアリソースをリアルタイムに監視できるアスキーUI（CUI）ユーティリティです。
Linux の古典的な監視ツール `nmon` にインスパイアされ、以下の要素を重視して設計されています。

- **ポータビリティ**: Windows、Linux、macOS などのクロスプラットフォーム環境で変更なしにビルド・動作すること。
- **超軽量性**: CPUやメモリなどのリソース自体の消費を最小限に抑え、監視対象システムに不要な負荷を与えないこと。
- **インタラクティブ性**: キー入力によって動的に表示セクションのトグル切り替えができること。

---

## 2. システムアーキテクチャ

本システムは単一の実行可能バイナリであり、単一スレッドのイベントループモデルを採用しています。

```text
┌────────────────────────────────────────────────────────┐
│                        MyNMON                          │
│                                                        │
│  ┌───────────────────────────┐                         │
│  │       Event Loop          │                         │
│  │  (1-second interval or)   │                         │
│  │  (Keyboard interrupt)     │                         │
│  └─────────────┬─────────────┘                         │
│                │                                       │
│                ▼                                       │
│  ┌───────────────────────────┐                         │
│  │   Metrics Collection      │                         │
│  │  (sysinfo crate)          │                         │
│  │  - CPU Cores, Memory      │                         │
│  │  - Disks, Networks, Proc  │                         │
│  └─────────────┬─────────────┘                         │
│                │                                       │
│                ▼                                       │
│  ┌───────────────────────────┐                         │
│  │     Terminal Rendering    │                         │
│  │  (crossterm crate)        │                         │
│  │  - Color styling          │                         │
│  │  - ASCII progress bar     │                         │
│  └───────────────────────────┘                         │
└────────────────────────────────────────────────────────┘
```

### 主要ライブラリと役割
- **`sysinfo`**: ハードウェア、プロセス、OS、ネットワーク、ディスクのメトリクスをOS非依存の抽象API経由で取得します。Windows APIおよびLinuxの `/proc` ファイルシステムなどのプラットフォーム固有の実装を内包しています。
- **`crossterm`**: 画面切り替え（オルタネートスクリーン）、カーソル非表示、rawモード制御、キーイベント入力の監視、およびターミナル制御シーケンスを実行します。

## 3. 機能要件

### 3.1 共通ヘッダー表示
画面最上部には、常にシステム基本情報と稼働状態が表示されます。画面崩れ（改行の折り返し）を防止するため、現在のターミナル幅（width）に応じて動的に表示項目を削減して描画を行います。
- **フル表示（横幅 105文字以上）**: プログラム名とバージョン（例: `MyNMON v0.4.0`） | Host: ホスト名 | OS: OS名 | Kernel: カーネルバージョン | Uptime: 起動時間 | Interval: リフレッシュ間隔
- **中縮小表示（横幅 90〜104文字）**: Kernelの情報を隠し、それ以外を表示。
- **小縮小表示（横幅 80〜89文字）**: KernelとOS名の情報を隠し、それ以外を表示。
- **システム起動時間 (Uptime)**: システムが起動してからの経過時間を `Xd HH:MM:SS` (1日未満の場合は `HH:MM:SS`) の形式で表示し、1秒ごとにリアルタイムに更新。
- **リフレッシュ間隔 (Interval)**: 画面の更新頻度（秒数）をヘッダーに表示（例: `Interval: 1s`）。

### 3.2 画面表示およびセクション制御

#### 3.2.1 起動時ウェルカム画面（初期ヘルプ）
初回起動時はすべての情報セクションが非表示（`false`）の状態で立ち上がります。
いずれのセクションも表示されていない時は、画面中央に「コマンド紹介」および「操作可能なキーボードショートカット一覧（ヘルプ）」が大きなレイアウトで描画されます。
いずれかのトグルキーを押してセクションをONにすると、この紹介画面は非表示になります。

#### 3.2.2 監視情報セクション
システムは以下の情報セクションを持ち、それぞれキー入力により動的に表示/非表示（トグル）を切り替えられます。

1. **CPU 全体セクション (`C` キー)**:
   - 全体のCPU使用率 (%) を取得し表示。
   - 使用率に応じたアスキー進行バー（`[===>    ]`）の描画。
   - 負荷レベルに応じた自動着色（緑色: <= 40%、黄色: 40% < 負荷 <= 80%、赤色: > 80%）。
2. **CPU コア別セクション (`c` キー)**:
   - 個々のCPUコアの使用率 (%) を取得し表示。
   - 使用率に応じたアスキー進行バー（`[===>    ]`）の描画。
   - 負荷レベルに応じた自動着色。
3. **メモリセクション (`m` キー)**:
   - 物理メモリ（RAM）およびスワップメモリ（Swap/Pagefile）の総容量、使用量、空き容量を GB 単位で表示。
   - 物理メモリとスワップメモリそれぞれの使用率 (%) を表示し、アスキーバーを描画（85%を超えると赤色に着色）。
4. **ディスクセクション (`d` キー)**:
   - 各ディスクのマウントポイント、ファイルシステム形式（例: `"NTFS"`, `"ext4"`）。
   - 空き容量 / 総容量、および使用率 (%) の表示。
5. **ネットワークセクション (`n` キー)**:
   - 稼働中の各ネットワークインターフェース名（辞書順にソートして描画）。
   - 受信速度（Rx）および送信速度（Tx）を KB/s 単位で表示。インターフェース名は全角半角の文字幅を正しく計算して固定幅（16幅）で綺麗にパディングされ、縦位置が整列されます。
6. **プロセスセクション (`p` または `t` キー)**:
   - CPU使用率が高い順にソートされた上位8件のアクティブプロセス。
   - PID（桁数に関わらず6桁右揃えでアライメント維持）、プロセス名（20幅に切り詰め/パディング）、CPU使用率 (%)、物理メモリ占有量 (MB) を表形式で美しく表示。
7. **プロセス検索フィルタ機能 (`f` キー)**:
   - `common_lib::count_occurrences` を使用し、入力された検索クエリが稼働中プロセス全体で何件マッチするかをカウント・表示。
   - プロセス一覧の表示を、検索クエリに前方/部分一致するプロセスのみに絞り込む。
   - `Enter` または `Esc` キーで検索入力モードを解除。
8. **プロセス起動・終了差分ログセクション (`g` または `l` キー)**:
   - `common_lib::compute_diff` を使用し、1ティック（設定されたリフレッシュ間隔）前のアクティブプロセス一覧と現在のプロセス一覧を比較。
   - 期間中に新しく追加（Added）されたプロセスを `+ 名前 (PID)` (緑色)、削除（Removed）されたプロセスを `- 名前 (PID)` (赤色) として時系列履歴（最大10件）で描画。
   - プロセス検索フィルタ（`f` キー）がアクティブな場合、差分ログ自体もフィルタリングされる。

### 3.3 アプリケーション制御キー
- `q` または `Esc` キーの入力により、ターミナルの状態（rawモード解除、オルタネートスクリーン離脱、カーソル再表示）を元通りに修復した上で、正常終了コード `0` で即座にプロセスを終了します。
- `r` キーの入力により、リフレッシュ秒数のインタラクティブ変更モードに入ります。数字を入力して `Enter` を押すと画面更新タイマーの秒数（tick_rate）が適用され、`Esc` でキャンセルされます（1秒以上の数値のみ有効）。

### 3.4 起動時オプション（コマンドライン引数）
アプリケーション起動時に引数を与えることで、以下の特殊な動作を実行して即時終了させることができます。

1. **ヘルプ表示 (`-h` または `--help`)**:
   - アプリケーションの使用方法、利用可能なオプション、および起動後の操作キーボードショートカットの一覧を表示し、正常終了します。
2. **バージョン表示 (`-v` または `--version`)**:
   - `Cargo.toml` の `version` フィールドからコンパイル時に解決された動的バージョン（例: `MyNMON v0.4.0`）を表示し、正常終了します。
3. **不正オプションのエラーハンドリング**:
   - 定義されていない引数が与えられた場合、標準エラー出力にエラーメッセージ（`Error: Unknown option '...'`）と使用方法（`Usage`）を出力し、終了ステータス `1` で異常終了します。

### 3.5 多重起動防止機能 (Windows専用)
- `common_lib::check_single_instance` を統合し、Windows上でのアプリケーションの多重起動を防止します。
- 起動時にシステム全域の Named Mutex（`MyNMON_NamedMutex_Instance`）を作成・確認し、既にインスタンスが稼働している場合は、標準エラー出力に `Error: Another instance of MyNMON is already running.` を表示し、終了ステータス `1` でプロセスを即座に正常終了します。

## 4. 非機能要件

- **更新頻度**: デフォルト1秒（1000ms）間隔でメトリクスをリフレッシュし画面を再描画します。更新間隔は `r` キーで動的に変更（秒単位）可能です。
- **キー割り込みレスポンス**: キー入力が発生した場合は、指定されたタイマー完了を待たずに即座に状態を変更して再描画を行います。
- **描画パフォーマンス (フリッカーレス & 画面保護)**:
  - 画面の最小解像度チェックとして **「横80文字以上、縦20行以上」** を制限とし、これを下回る場合は画面崩れ防止のため、警告メッセージのみを表示して描画をスキップ（保護）します。
  - 画面更新の際、画面全体をクリアする（`Clear(All)`）のを避け、カーソルを左上に固定して上書き描画を行います。これにより、CUI特有の激しい画面のちらつきを排除しています。
  - 各行の描画時には「行末までクリア（`UntilNewLine`）」を適用し、描画完了後に「画面下部をクリア（`FromCursorDown`）」を適用することで、表示情報の縮小時にも古いデータが残らないように制御されています。
- **フットプリント**:
  - **実行ファイルサイズ**: 350 KB 以下（最適化リリースビルド時）。
  - **実行時メモリ使用量**: 25 MB 以下（通常のプロセス監視時）。
- **ロバスト性**:
  - パニック発生時はcrosstermによる画面破壊を避けるため、即座にプロセスを中断（`panic = 'abort'`）します。
  - OS情報の取得失敗やディスクが存在しない場合でも、フォールバック値（例: `"Unknown"`, `0`）を用いてプログラムがクラッシュせずに処理を続行します。
