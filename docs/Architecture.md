# システム設計書 (Architecture.md)

本ドキュメントは、クロスプラットフォーム対応・超軽量CUIシステムモニター `MyNMON` のシステム設計、技術スタック、ディレクトリ構造の意図、およびモジュール間のデータ連携について定義したものです。

---

## 1. システム概要と目的

`MyNMON` は、Windows、Linux、macOS 環境に対応した超軽量なコマンドライン（CUI）システムモニターです。クラシックなLinuxシステムモニターツールである `nmon` にインスパイアされており、以下の目的を持って開発されています。

- **超軽量動作と低リソース消費**:
  監視ツール自体が大量のリソースを消費してシステムに余計な負荷を与えるのを防ぐため、CPU負荷を最小限にし、実行ファイルサイズ（約 300KB）および動作メモリ（約 20MB）を極めて低く抑えます。
- **クロスプラットフォーム対応**:
  Windows、Linux、macOS でコードを変更することなく同一のバイナリをビルド・実行可能にします。
- **インタラクティブなセクション制御**:
  ターミナル上で特定のキーを入力することで、CPU、メモリ、ディスク、ネットワーク、プロセスなどの監視セクションを瞬時にトグル切り替え（表示/非表示）できます。

---

## 2. 採用している技術スタック

本システムは Rust 言語で開発されており、以下の主要ライブラリ・フレームワークを使用しています。

| ライブラリ名 | バージョン | 主要な役割・採用の意図 |
| :--- | :--- | :--- |
| **Rust** | 1.96.0+ (Edition 2021) | 高い実行パフォーマンス、メモリ安全性、およびクロスプラットフォームの単一バイナリ生成のため。 |
| **sysinfo** | 0.30.12 | OS非依存の抽象APIを介して、CPU、メモリ、ディスク、ネットワーク、プロセスのメトリクスを取得するため。プラットフォーム固有の実装（`/proc` や Windows API）を安全に隠蔽します。 |
| **crossterm** | 0.29.0 | ターミナルの生モード（raw mode）制御、オルタネートスクリーンへの切り替え、カーソルの表示/非表示、およびキーボード入力イベントの非同期ポーリングのため。 |
| **common_lib** | ローカル参照 | プロジェクト共通のユーティリティ。Windowsでの二重起動防止（Named Mutex）、プロセス一覧の差分検知（`compute_diff`）、および文字列マッチ数カウント等を提供。 |

## 3. ディレクトリ構造とその意図

プロジェクトは以下のように整理されており、機能ごとに役割を明確に分担しています。

```text
MyNMON/
├── .agents/             # AIエージェント設定・指示ディレクトリ
│   ├── AGENTS.md        # プロジェクト共通のAI開発ルール・規約
│   └── Instructions.md  # コーディングスタイル・AI制約（新規）
├── docs/                # システムドキュメント・各種設計書
│   ├── DIAGRAM.md       # イベントループ・データフロー図 (Mermaid)
│   ├── FOOTPRINTS.md    # バイナリサイズ・メモリ使用量の計測記録
│   ├── SPEC.md          # システム仕様書 (要件定義)
│   ├── TEST_REPORT.md   # ビルドおよび手動・自動テスト報告書
│   ├── Architecture.md  # システム設計書 (本ファイル)
│   └── Todo.md          # 開発ロードマップ・タスク管理（新規）
├── src/                 # ソースコードディレクトリ
│   ├── main.rs          # アプリケーションのエントリポイント & メインループ
│   ├── state.rs         # アプリケーション内部状態 (MonitorState) の定義
│   ├── ui.rs            # crossterm を使用したターミナル描画ロジック
│   └── utils.rs         # フォーマット変換・文字列操作用ユーティリティ
├── Cargo.toml           # パッケージ構成 & リリース最適化設定
├── CHANGELOG.md         # 開発・修正履歴の記録ファイル
└── README.ja.md / .md   # プロジェクトの概要と使用方法 (日/英)
```

### 各ソースファイルの役割
1. **[main.rs](../src/main.rs)**:
   アプリケーションの起点。二重起動の検知、コマンドラインオプション解析、crosstermによる画面初期設定、およびメインのイベントループ（1秒間隔の定期更新とキーイベント監視）を管理します。
2. **[state.rs](../src/state.rs)**:
   表示状態のトグル（どのセクションを表示するか）、フィルタリング用の検索クエリ、およびプロセス変化ログなどを保持する `MonitorState` 構造体を定義します。
3. **[ui.rs](../src/ui.rs)**:
   crossterm を使用した画面出力エンジン。フリッカー（画面のちらつき）を完全に防止するため、全画面クリアではなくカーソルを左上に戻す上書き描画を行います。
4. **[utils.rs](../src/utils.rs)**:
   Uptimeの文字フォーマット変換、ASCIIプログレスバー生成、および全角・半角を考慮した文字列パディング処理などの計算関数を提供します。

## 4. データフローとモジュール間の連携

本システムはシングルスレッドのイベントループモデルで動作しており、データ取得から描画、ユーザー入力処理までを1つのループで処理します。

```text
    +--------------------------------------------------------+
    |                      main.rs                           |
    |  1. 二重起動防止 (common_lib)                          |
    |  2. crossterm 初期設定 (オルタネートスクリーンなど)    |
    +--------------------------------------------------------+
                               |
                               v
                     +------------------+
                     |  Event Loop      | <------------------+
                     +------------------+                    |
                               |                             |
                               | (1s タイマー / キー割り込み)|
                               v                             |
    +--------------------------------------------------------+       |
    |               Metrics Collection                       |       |
    |  - sysinfo で CPU, RAM, Disk, Net, Proc データを取得   |       |
    |  - common_lib で前回プロセスとの差分検知 (Added/Removed) |       |
    +--------------------------------------------------------+       |
                               |                             |
                               v                             |
    +--------------------------------------------------------+       |
    |               State & UI Rendering                     |       |
    |  - MonitorState のトグル値に従って描画セクションを選択  |       |
    |  - ui::draw_ui が crossterm でオルタネート画面に描画    |       |
    +--------------------------------------------------------+       |
                               |                             |
                               v                             |
    +--------------------------------------------------------+       |
    |               Keyboard Event Poll                      |       |
    |  - crossterm::event::poll でキー入力を待機             |       |
    |  - キー入力あり: 入力されたキーに応じて State をトグル  |-------+
    |  - タイムアウト: そのまま次ループへ                    |
    +--------------------------------------------------------+
                               | ('q' or 'Esc' 入力)
                               v
    +--------------------------------------------------------+
    |                      Clean Up                          |
    |  - rawモードの解除、通常画面への復帰、カーソル再表示   |
    |  - プロセスの正常終了 (Ok)                             |
    +--------------------------------------------------------+
```

### 主要な連携処理
- **プロセス差分検知の流れ**:
  1. `main.rs` はループごとに `sysinfo::System::processes()` を取得。
  2. プロセスIDとプロセス名を `PID:名前` の改行区切りテキストに変換。
  3. `common_lib::compute_diff` に前回のテキストと現在のテキストを渡す。
  4. 差分 (`Added`, `Removed`) が検出された場合、起動・終了ログ（例: `+ cargo (PID: 1234)`）を `MonitorState.spawn_exit_log` に追加。
- **フリッカーレス描画の流れ**:
  1. `ui::draw_ui` は描画の開始時に `cursor::MoveTo(0, 0)` を実行。
  2. `w_line!` マクロにより、各行を書き出した直後に `Clear(UntilNewLine)` を実行し、古い残存文字をクリアしながら上書き。
  3. 描画終了時に `Clear(FromCursorDown)` を実行して、表示要素が減った場合でも下部の古い情報をクリア。これにより、画面全体を一瞬クリアする処理が不要になり、フリッカー（ちらつき）のない滑らかな描画を実現。
