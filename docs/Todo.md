# タスク管理書 (Todo.md)

本ドキュメントは、CUIシステムモニター `MyNMON` の開発状況（現在地）と、今後のロードマップを管理するためのタスク管理ファイルです。

---

## 1. 実装済みの機能 (Done)

これまでに実装が完了し、動作検証が行われた機能の一覧です。

- **CUI システムモニターの基本フレームワーク**:
  - `crossterm` を用いたオルタネートスクリーンへの切り替え、カーソル非表示、rawモード制御。
  - シングルスレッドのイベントループモデルの実装。
  - `q` または `Esc` によるクリーンな終了処理（ターミナルの復元）。
- **リアルタイムリソース監視セクション**:
  - **CPU全体**: 使用率 (%) の表示と、負荷に応じたカラーリング（緑・黄・赤）およびASCII進行バー（`C` キー）。
  - **CPU個別コア**: 各コアの使用率と個別の進行バー表示（`c` キー）。
  - **メモリ**: 物理メモリ（RAM）およびスワップメモリ（Swap/Pagefile）の容量、使用率、進行バー表示（`m` キー）。
  - **ディスクマウント**: 各ディスクのマウントポイント、ファイルシステム形式、容量・使用率の表示（`d` キー）。
  - **ネットワーク**: 各インターフェースのデータの受信・送信速度を KB/s 単位で表示（`n` キー）。
  - **プロセス一覧**: CPU使用率の高い順にソートされた上位8プロセスの表示（`p`/`t` キー）。
- **高度なインタラクティブ機能**:
  - **プロセス検索/フィルタ**: `f` キーにより、プロセス名で一覧表示をリアルタイムに絞り込み。マッチ件数も表示。
  - **プロセス起動・終了の動的ログ**: `common_lib::compute_diff` を利用し、1ティック前のアクティブプロセス一覧と比較して起動（`+`）または終了（`-`）したプロセスを検知・表示（`g`/`l` キー）。
  - **更新間隔の動的変更**: `r` キーにより、画面のリフレッシュレート（tick_rate）を任意の秒数に変更可能。
- **品質・ロバスト性対策**:
  - **画面崩れ・折り返し防止**: ターミナルサイズが 80x20 未満の場合に描画をスキップして警告を表示する安全装置。
  - **ちらつき（フリッカー）防止**: 画面の全体クリアを行わず、カーソルを左上に戻す上書き描画 (`MoveTo(0,0)`) と、行末クリア (`UntilNewLine`) / 画面下部クリア (`FromCursorDown`) の組み合わせによるフリッカーレス化。
  - **Windows専用二重起動防止**: `common_lib::check_single_instance` を統合し、Named Mutex による多重起動防止。
  - **リリースサイズ最適化**: `Cargo.toml` におけるサイズ最適化 (`opt-level = 'z'`, `lto = true`, `codegen-units = 1`, `strip = true`) の適用。

## 2. 直近のタスク (Todo / In Progress)

ソースコードや仕様書から、現在不足している、あるいは改善すべき直近のタスクです。

- `[ ]` **更新間隔の小数点（ミリ秒）対応**:
  - 現在 `r` キーでのリフレッシュ間隔変更は `u64` (秒単位) であり、1秒未満（例: 0.5秒）やミリ秒単位での細かな調整ができない。これを `Duration` で扱えるようにし、浮動小数点数（例: `0.5`, `1.5`）での入力を可能にする。
- `[ ]` **プロセス差分履歴ログのクリア・スクロール機能**:
  - 現在のプロセス差分ログ（`g`/`l` キー）は自動的に最大50件（表示は最新10件）でリングバッファのように古いものが切り捨てられるが、手動で履歴をクリアする機能や、過去のログをスクロールして確認する手段がない。
- `[ ]` **自動テスト（Unit Test）の追加・カバレッジ向上**:
  - 現在自動テストは `utils::get_ascii_bar` の検証1件のみである。`utils::pad_or_truncate` や `utils::format_uptime`、また `utils::parse_proc_line` などの他の純粋関数のテストを追加し、テストによる品質担保を強化する。
- `[ ]` **Clippyおよびコンパイル警告のゼロ維持**:
  - 今後の機能追加や Rust ツールチェーンのアップデート時に、`cargo clippy` で警告が発生しないクリーンな状態を維持する。

## 3. 将来的な拡張提案 (Backlog)

さらにアプリケーションを強力にするための長期的な機能追加のアイデアです。

- `[ ]` **macOS / Linux 等の他プラットフォームでの二重起動防止**:
  - 現在 `common_lib::check_single_instance` は Windows 専用の Named Mutex を用いている。Unix系 OS 向けにファイルロック（例: `/var/run/MyNMON.pid` の作成や `flock`）を用いた二重起動防止処理を `common_lib` に追加し、クロスプラットフォームで二重起動防止を実現する。
- `[ ]` **詳細なネットワークメトリクスの追加**:
  - 現在の Rx/Tx 速度 (KB/s) に加えて、積算転送量 (MB) やパケット数、エラー数を切り替えて表示できる機能。
- `[ ]` **ディスクI/O速度（読込/書込速度）の表示**:
  - 現在ディスクはマウント情報と空き容量のみの表示だが、ディスクへの動的なアクセス速度（Read/Write KB/s）を `sysinfo` から取得して表示するセクションを追加する。
- `[ ]` **GPU / VRAM 監視セクションの追加**:
  - システムに搭載されている GPU（NVIDIA, AMD 等）の使用率や、VRAM（ビデオメモリ）の割り当て状況を表示するセクションを追加。
- `[ ]` **プロセス一覧のソート基準切り替え機能**:
  - 現在プロセス一覧は CPU 使用率順で固定されているが、これを `m` キー等で「メモリ使用量順」に切り替えてソートできる機能。
- `[ ]` **UIカラーテーマのカスタマイズ**:
  - モノクロ、デフォルト（カラフル）、ダーク、ライトなど、ユーザーの好みに応じた表示色のテーマ切り替え機能。
