# フットプリント測定レポート (FOOTPRINTS.md)

本レポートは、リリースビルドにおける `rust-nmon` のバイナリサイズおよびメモリ使用量の実測値を記録したものです。

---

## 1. 測定環境

- **OS**: Windows 11 Pro (ビルド 26200)
- **CPU**: Intel(R) Core(TM) i7 / 12スレッド
- **RAM**: 16 GB
- **Rust バージョン**: rustc 1.78.0 (または最新の stable ツールチェーン)
- **Cargo プロファイル設定 (Cargo.toml)**:
  ```toml
  [profile.release]
  opt-level = 'z'       # サイズ優先最適化
  lto = true            # リンク時最適化
  codegen-units = 1     # コンパイル単位の統合
  panic = 'abort'       # 巻き戻し無効化
  strip = true          # シンボル情報の削除
  ```

---

## 2. 測定結果

### 2.1 実行バイナリサイズ
最適化を施したリリースプロファイルでビルドされた実行ファイルのサイズです。

- **ファイルパス**: `target/release/rust-nmon.exe`
- **ファイルサイズ**: **306,688 バイト (約 299 KB)**

> [!TIP]
> リリース最適化設定を行わないデフォルトのデバッグビルド（`target/debug/rust-nmon.exe`）では通常数メガバイト規模になりますが、最適化により 300 KB を下回るサイズまで圧縮されています。

### 2.2 実行時メモリフットプリント
実際にアプリケーションを起動し、2秒間稼働させた時点でのプロセス物理メモリ使用量（Working Set）を測定しました。

- **プロセス名**: `rust-nmon.exe`
- **物理メモリ使用量 (WorkingSet64)**: **18,694,144 バイト (約 17.8 MB)**

> [!NOTE]
> アプリケーション内部の `sysinfo` クレートが内部バッファに全プロセスのリストやハードウェア統計をキャッシュしているため、17 MB 程度のメモリ空間を確保しています。これはCUIシステムモニターとしては十分に軽量であり、他のアプリケーションのパフォーマンスを阻害しません。

---

## 3. 総評

`rust-nmon` は、非常にコンパクトなバイナリサイズ (約 299 KB) と、低負荷・低メモリ消費 (約 17.8 MB) を実現しています。
これにより、サーバー監視用途やリソースが極めて制限された環境でも、問題なくバックグラウンド動作またはフォアグラウンドでのリアルタイム監視に使用することができます。
今後機能を追加する際も、このフットプリント水準（バイナリ 350 KB 以下、メモリ 25 MB 以下）を維持するよう努めます。
