# システム設計書 (ARCHITECTURE.md)

[English](../en/ARCHITECTURE.md) | **日本語版**

本ドキュメントは、超軽量 CUI システムモニター `MyNMON` のアーキテクチャ設計、コンポーネント構成、データフロー、および条件付きコンパイル（Cargo Features）によるモジュール分離方針を定義した設計書です。

---

## 1. 全体構成およびモジュール分割

`MyNMON` は関心の分離に基づき、以下の 4 つのソースファイルに分割されています。

```text
src/
├── main.rs      # エントリポイント、二重起動チェック、CLI引数解析、メインイベントループ
├── state.rs     # アプリケーション状態構造体 MonitorState の定義 (feature制御)
├── ui.rs        # Crossterm を用いたターミナル描画関数群 (feature制御)
└── utils.rs     # 時間フォーマット、ASCIIバー生成、マルチバイト文字幅計算等のユーティリティ
```

---

## 2. Cargo Features（条件付きコンパイル）アーキテクチャ

用途や組み込み環境に応じたバイナリサイズの最適化のため、各監視モジュールを Feature フラグとして分離しています。

```text
               ┌─────────────────────────────┐
               │    default = ["cpu", ...]   │
               └──────────────┬──────────────┘
                              │
  ┌──────────┬──────────┬─────┴────┬──────────┬──────────┐
  │          │          │          │          │          │
  ▼          ▼          ▼          ▼          ▼          ▼
[cpu]      [mem]      [disk]     [net]      [proc] ◄── [diff]
```

- 各機能 (`cpu`, `mem`, `disk`, `net`, `proc`, `diff`) のデータ取得・描画ロジック・キー操作イベントは、`#[cfg(feature = "...")]` によりコンパイル時に組み込み/除外されます。
- `diff`（プロセス増減ログ）機能はプロセス情報の比較を行うため、`proc` feature に自動的に依存します。
