# セキュリティポリシー (SECURITY.md) - MyNMON

[English](../en/SECURITY.md) | **日本語版**

本文書は、`MyNMON` プロジェクトにおけるセキュリティ方針および脆弱性報告手順をまとめたドキュメントです。

---

## 1. セキュリティ設計と安心性

`MyNMON` は以下の設計原則により、高いセキュリティと信頼性を保証しています。

1. **メモリ安全性**:
   - Rust 言語の強力な所有権システムと型安全性を活用し、メモリ安全性上の脆弱性（バッファオーバーフロー、ヌルポインタ参照等）を排除します。
2. **ローカル完結型の動作**:
   - ネットワークの監視（受信/送信速度の表示）は行いますが、外部サーバーとの通信は一切行わず、純粋にローカル環境のシステム状態取得とコンソール表示のみを行います。情報の外部送信等は行いません。
3. **安全なシャットダウン**:
   - シグナルやキーボードによる終了時、ターミナルモード（rawモード）を正常に解除して復元するようクリーンアップ処理を徹底しています。

| :--- | :---: |

---

## 3. 脆弱性の報告方法

`MyNMON` に潜在的なセキュリティ脆弱性を発見された場合は、公開 Issue を作成するのではなく、以下の手順でご報告ください。

1. **報告窓口**:
   - リポジトリの管理者に直接ご連絡いただくか、セキュリティ報告用窓口へメールを送信してください。
2. **記載いただきたい内容**:
   - 影響を受ける `MyNMON` のバージョンとOS環境
   - 脆弱性の詳細と再現手順（概念実証コードまたはコマンド例）
3. **対応プロセス**:
   - 報告を受領後、3日以内に受領確認を行い、修正パッチを作成・検証したうえで速やかにリリースを行います。
